iOS 开发相关

1) 原生 iOS(Swift) 和 React-Native 的性能比较
• 作者简介：John A. Calderaio(原) Deepmissea(译)
• 内容简介：我的目标是找出他们是否能够准确地履行他们的承诺。要实现目标的话，我就需要用 Swift 和 React-Native 构建相同的应用。它需要足够简单，以便我可以学习两种语言并及时完成应用程序，但也需要足够的复杂，才能比较每个应用的 CPU、GPU、内存的使用情况和功耗。
• 个人点评：之前我一直都困扰于 RN 的性能是否能跟得上原生，因为也只是搭起来自己随便玩了一阵子，没有做过项目，结论还是无从得知。但是从文章的测试结果来看，RN 的性能貌似是比原生的好，且不说测试结果的可靠性有多大，但是 RN 如果能跟原声的效率不相上下，那 RN 也算是非常成功了。实际项目虽说不可能完全推到转到RN，但是完全可以在某些模块用 RN 去实现，毕竟它的动态更新和跨平台还是非常吸引的。

• 作者简介：Joy
• 内容简介：子线程默认不会开启 Runloop，那出现 Autorelease 对象如何处理？不手动处理会内存泄漏吗？

3) 深入理解RunLoop
• 作者简介：YY
• 内容简介：RunLoop 是 iOS 和 OSX 开发中非常基础的一个概念，这篇文章将从 CFRunLoop 的源码入手，介绍 RunLoop 的概念以及底层实现原理。之后会介绍一下在 iOS 中，苹果是如何利用 RunLoop 实现自动释放池、延迟回调、触摸事件、屏幕刷新等功能的。

4) iOS最全性能优化(上) iOS最全性能优化(中) iOS最全性能优化(下)
• 作者简介：HelloYeah
• 内容简介：性能对 iOS 应用的开发尤其重要，如果你的应用失去反应或者很慢，失望的用户会把他们的失望写满App Store的评论。然而由于iOS设备的限制，有时搞好性能是一件难事。开发过程中你会有很多需要注意的事项，你也很容易在做出选择时忘记考虑性能影响。

5) Objective-C 的运行时以及 Swift 的动态性
• 作者简介：Roy Marmelstein
• 内容简介：自 Swift 推出以来，人们总是不断希望 Swift 能够「更加动态化」。但是「动态化 (dynamic)」又是什么意思呢？为什么 Objective-C 比 Swift 更加动态化呢？如果我们无法使用动态化的语言，那么该如何去构建以往那些依赖于动态性的东西呢？在本次 Mobilization 2016 大会上，Roy Marmelstein 详细阐述了这些问题。

A fast, simple & powerful blog framework

Hexo 是一个轻量级、基于 Node.js 的静态博客框架，生成的静态网页可以托管在 Github 或者 HeroKu 上。

What we need？

Github Pages

Github Pages 可以被认为是用户编写的、并且托管在 Github 上的静态网页。GP 生成的网站的默认域名是 username.github.io 或者是 username.github.io/project-name，同时，GP也支持自定义域名,不过本文暂不介绍，使用的是默认域名。

Mac

本博客搭建过程使用的是 Mac 下的 小黑框，Linux 应该也是一样的，windows 就不考虑了-。-

Markdown

我们只需要到 _posts 名录下修改 xxx.md 文件即可，修改完成后，执行 hexo g 然后 hexo s 在本地确认没问题后执行 hexo d 即可推送到 Github。
